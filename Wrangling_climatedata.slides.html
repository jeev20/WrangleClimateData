Data formats¶

1. GRIB - GRIdded Binary or General Regularly-distributed Information in Binary form¶

+ Smaller file sizes
+ Open for data manipulation when combining multiple files

2. Netcdf4 - Network Common Data Format¶

NetCDF is a data format that support the creation, access, and sharing of array-oriented scientific data. This format is mainly used by the Climate Community. 

+ Many modules support netcdf format as of 2019 
- Large file sizes depending on number of parameters

3. HDF - Hierarchical Data Format¶

Software library and file format to manage, process, and store your heterogeneous data. 

+ Light weight data format - used extensively in development of Neural networks
- Relatively low use in the climate analysis communities

Services / interfaces¶

1. THREDDS - Thematic Real-time Environmental Distributed Data Services¶

A web server that provides metadata and data access for scientific datasets, using a variety of remote data access protocols.

2. OPENDAP - Open-source Project for a Network Data Access Protocol¶

OPeNDAP provides software which makes local data accessible to remote locations regardless of local storage format. 

+ User friendly

3. WMS - Web map service¶

- XML tree parsing is expensive and time consuming

4. WCS - Web coverage service¶

- XML tree parsing is expensive and time consuming

import requests
import xmltodict

lat = round(62.628999, 2)
lon = round(6.662294, 1)

# Query the /weatherapi/oceanforecast/ api with latitude and longitude values
url = "https://api.met.no/weatherapi/oceanforecast/0.9/?lat="+str(lat)+"&lon="+str(lon)
print (url)
r = requests.get(url)
doc = xmltodict.parse(r.content)
x = doc['mox:Forecasts']['mox:forecast']
depth = x[0]['metno:OceanForecast']['mox:seaBottomTopography']['#text']

4. Plotting grid-based data¶

Grid based data can

+ be large and hard to contemplate
+ have different projections
+ have different lattitude and longitude conventions 
+ require working with vector fields

Plotting the data helps in

+ understanding the data and its trends visually
+ validating if you choose the correct location or not

Plotting tools¶

1. Panoply is a light-weight java based visualizer for grid-based data

+ Excellent tool to get an overview of the data
+ GUI interface and supports GRIB, NETCDF, HDF formats
- Cannot manipulate data to build custom maps / plots

2. Basemap is based on the standard python plotting library Matplotlib

+ Maps can be customized
+ Very good documentation
- A steep learning curve
- Can have installation issues

3. Geopandas a python library for plotting grid-based data

+ Maps / plots can be customized
+ Very easy to learn and build code
- Not as customizable as basemap

Helper functions to ease data manipulation tasks¶

In [ ]:
def time_string(time):
    """
    returns the time variable in the required format
    """
    #return datetime.utcfromtimestamp(time).strftime('%d-%m-%Y at %H:%M:%S')
    return pd.Timestamp(float(time)).strftime('%d-%m-%Y')

def geo_idx(dd, dd_array):
   """
     search for nearest decimal degree in an array of decimal degrees and return the index.
     np.argmin returns the indices of minium value along an axis.
     so subtract dd from all values in dd_array, take absolute value and find index of minium.
    """
   geo_idx = (np.abs(dd_array - dd)).argmin()
   return geo_idx
